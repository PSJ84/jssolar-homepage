# js솔라 블로그 구축 - jssolar.kr/blog

## 프로젝트 배경 및 목적

js솔라는 1인 태양광 EPC/인허가/O&M 전문기업이다.
이 블로그의 핵심 목적은 두 가지다:

### 1. AI 시대 검색 최적화 (AEO/GEO)
- 기존 SEO를 넘어, AI 답변 엔진(Google AI Overview, ChatGPT, Perplexity)에서 js솔라 콘텐츠가 인용되는 것이 목표
- 구조화된 데이터(JSON-LD)를 통해 AI가 "자격 있는 전문가가 쓴 신뢰할 수 있는 콘텐츠"로 인식하게 함
- E-E-A-T(경험, 전문성, 권위, 신뢰) 신호를 극대화

### 2. 고객 신뢰 확보 + 업계 전문성 어필
- 시공 사례와 기술 콘텐츠를 혼합하여 운영
- 태양광 현장 실무 경험 기반의 전문 콘텐츠 제공
- 잠재 고객이 블로그를 통해 js솔라의 역량을 확인할 수 있게 함

---

## 기술 요구사항

### 스택
- 기존 jssolar.kr과 동일한 Next.js 프로젝트 내에서 구현
- 경로: `/blog` (서브도메인 아님, 같은 도메인 내 서브디렉토리)
- 콘텐츠: MDX 또는 마크다운 파일 기반 (CMS 없이 파일로 관리)
- 정적 생성(SSG) 방식으로 빌드

### 디자인
- **기존 jssolar.kr 홈페이지의 디자인 시스템(색상, 폰트, 레이아웃 스타일)을 그대로 따를 것**
- 기존 사이트의 헤더/네비게이션에 "블로그" 메뉴 항목 추가
- 반응형 (모바일 우선)
- 깔끔하고 읽기 편한 레이아웃

### 블로그 목록 페이지 (`/blog`)
- 카테고리 필터 (전체 / 기술 해설 / 시공 사례 / 규정 정리 / O&M)
- 각 글 카드에 표시할 정보: 카테고리, 제목, 요약, 날짜, 읽기 시간
- 작성자 표시: **"성진 · js솔라 대표"** (전기기사라는 표현은 화면에 표시하지 않음)

### 블로그 상세 페이지 (`/blog/[slug]`)
- MDX 렌더링 (코드 블록, 테이블, 이미지 지원)
- 목차(TOC) 자동 생성
- 이전/다음 글 네비게이션
- 작성자 프로필 영역: 이름, 직함(js솔라 대표), 간단한 소개

### RSS 피드
- `/blog/rss.xml` 경로로 RSS 피드 생성

## JSON-LD 구조화 데이터 (핵심)

모든 블로그 글에 아래 JSON-LD를 `<head>`에 자동 삽입할 것.
이것이 이 블로그의 가장 중요한 기술적 요소다.
AI와 검색엔진이 이 데이터를 읽고 전문성과 신뢰도를 판단한다.

**화면에는 "js솔라 대표"만 표시하고, 전기기사 자격 정보는 JSON-LD에만 포함한다.**

### BlogPosting 스키마 (각 글마다)
```json
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "글 제목 (frontmatter에서 가져옴)",
  "description": "글 요약 (frontmatter에서 가져옴)",
  "datePublished": "발행일",
  "dateModified": "수정일",
  "image": "대표 이미지 URL (있을 경우)",
  "author": {
    "@type": "Person",
    "name": "성진",
    "jobTitle": "js솔라 대표",
    "url": "https://jssolar.kr",
    "hasCredential": {
      "@type": "EducationalOccupationalCredential",
      "credentialCategory": "전기기사",
      "recognizedBy": {
        "@type": "Organization",
        "name": "한국산업인력공단"
      }
    },
    "knowsAbout": ["태양광 EPC", "태양광 O&M", "태양광 인허가", "전기설비", "계통연계"],
    "worksFor": {
      "@type": "LocalBusiness",
      "name": "js솔라",
      "url": "https://jssolar.kr",
      "description": "태양광 EPC, 인허가, 컨설팅, O&M 전문기업",
      "areaServed": "대한민국",
      "serviceType": ["태양광 EPC", "태양광 인허가", "태양광 O&M", "태양광 컨설팅"]
    }
  },
  "publisher": {
    "@type": "Organization",
    "name": "js솔라",
    "url": "https://jssolar.kr"
  },
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "해당 글 URL"
  }
}
```

### Blog 목록 페이지용 스키마
```json
{
  "@context": "https://schema.org",
  "@type": "Blog",
  "name": "js솔라 블로그",
  "description": "태양광 EPC·O&M 전문가가 직접 쓰는 현장 실무 블로그",
  "url": "https://jssolar.kr/blog",
  "author": {
    "@type": "Person",
    "name": "성진"
  }
}
```

## MDX 글 Frontmatter 형식

```yaml
---
title: "글 제목"
description: "글 요약 (1-2문장)"
date: "2026-03-15"
updated: "2026-03-15"
category: "기술 해설"  # 기술 해설 | 시공 사례 | 규정 정리 | O&M
tags: ["KEPCO", "계통연계", "500kW"]
thumbnail: "/blog/images/xxx.jpg"  # 선택
readingTime: 8  # 분 단위, 자동 계산이면 더 좋음
---
```

## 주의사항

- 기존 jssolar.kr 코드를 먼저 파악한 후, 기존 디자인 시스템에 맞춰서 구현할 것
- 기존 코드/기능은 절대 깨뜨리지 말 것
- 불필요한 외부 라이브러리는 최소화 (MDX 처리용 라이브러리만 추가)
- 성능: 정적 생성으로 빠른 로딩 유지
- 접근성: 시맨틱 HTML, 적절한 heading 구조
